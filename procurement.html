Penjualan

Periode Penjualan A & B

Setiap bulan terbagi menjadi dua periode dengan rentang tanggal tetap.

Periode A

1 - 15

Berjalan setiap tanggal 1 sampai 15 tanpa perlu memilih bulan atau tahun.

Periode B

16 - 30/31

Mulai tanggal 16 hingga akhir bulan, menyesuaikan jumlah hari di bulan tersebut.

Periode B berlangsung dari tanggal 16 sampai akhir bulan.

Supabase

Edge Function auto-procurement

Fungsi akan membuat baris di procurement_needs ketika tanggal WIB hari ini sama dengan tanggal mulai periode - lead_time pada katalog produk.

POST https://<project-ref>.functions.supabase.co/auto-procurement

Cron dijadwalkan setiap 00:00 WIB (17:00 UTC) melalui supabase/cron.json.

  • Memakai zona waktu Asia/Jakarta.
  • Melewati SKU yang sudah memiliki target_date sama pada tabel pengadaan.
  • Menandai prioritas sebagai tinggi dengan status direncanakan.

SQL Editor

Katalog Lead Time

Jalankan skrip berikut di Supabase SQL Editor untuk membuat tabel sumber otomatisasi.

-- Lihat file supabase/procurement_auto.sql
create table if not exists procurement_catalog (
  id uuid primary key default uuid_generate_v4(),
  product_name text not null,
  sku text not null unique,
  lead_time_days integer not null default 0,
  default_vendor text,
  default_quantity numeric not null default 0,
  unit text not null default 'pcs',
  is_active boolean not null default true,
  notes text,
  created_at timestamptz default timezone('utc', now()),
  updated_at timestamptz default timezone('utc', now())
);

Tambahkan baris contoh untuk mengetes otomatisasi: ('VR Headset', 'SKU-VR-001', 5, 'Vendor A', 25, 'pcs').
